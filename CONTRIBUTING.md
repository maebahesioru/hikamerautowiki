# コントリビューション

## Issue（課題）

- **バグ:** 起きたこと・期待したこと、**再現手順**（コマンドや画面操作）、**環境**（OS、Node のバージョン、`pnpm` のバージョン、ブランチやコミットが分かれば記載）。API キーやパスワード、`.env.local` の内容は **書かない**（マスクまたは「設定済み」とだけ書く）。
- **機能要望・質問:** 使いたい場面や、既存の README / `help` ページで足りない点があれば併記すると議論しやすいです。

テンプレートから作成すると項目が揃います（GitHub の **Issues → New issue**）。

## Pull Request（修正の提案）

1. **フォーク**してブランチを切るか、このリポジトリに書き込み権限がある場合はトピック用ブランチを切る。
2. 変更後、ローカルで次が通ることを確認する（CI と同じです）。

   ```bash
   pnpm install
   pnpm lint
   pnpm test
   pnpm build
   ```

3. PR の説明に **目的・変更内容の要約**、関連する **Issue 番号**（`#123` のように）があれば書く。
4. **秘密情報をコミットしない**（`.env*` は `.gitignore` 対象。誤って追加した場合はコミット前に `git reset` や履歴修正が必要になることがあります）。

`master`（または `main`）向けの PR では、GitHub Actions の **CI**（lint / test / build）が自動で走ります。
